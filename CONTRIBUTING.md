# Contributing to Lineage Mapper

Thank you for helping us improve Lineage Mapper! This document provides guidelines for contributing to the project.

## How to Contribute

### Reporting Bugs
- Check existing issues before reporting
- Provide a clear description of the bug
- Include steps to reproduce
- Mention your browser and OS
- Attach screenshots if possible

### Suggesting Features
- Open an issue with the `enhancement` label
- Describe the use case and benefits
- Explain how it improves the genealogy experience
- Include mockups or examples if helpful

### Submitting Pull Requests
1. Fork the repository
2. Create a feature branch (`git checkout -b feature/your-feature`)
3. Make your changes with clear commit messages
4. Test thoroughly in multiple browsers
5. Push to your branch
6. Open a Pull Request with detailed description

## Code Style Guidelines

### HTML
- Use semantic HTML elements
- Add descriptive IDs and classes
- Include ARIA labels for accessibility
- Keep structure clean and organized

### CSS
- Use consistent naming conventions
- Group related styles with comments
- Keep mobile-first approach
- Maintain responsive breakpoints

### JavaScript
- Use meaningful variable and function names
- Add comments for complex logic
- Keep functions focused and modular
- Use ES6+ features where appropriate
- Include error handling

## Testing Before Submission
- Test on Chrome, Firefox, Safari, and Edge
- Test on mobile and tablet devices
- Verify data export/import functionality
- Check accessibility with keyboard navigation
- Test with various family tree sizes

## Development Setup
1. Clone the repository
2. Make changes to source files
3. Open `index.html` in a browser to test
4. Use browser developer tools for debugging

## Data Format Standards
- Use ISO 8601 dates (YYYY-MM-DD)
- Maintain consistent naming conventions
- Follow existing JSON structure for exports
- Document any new data fields

## Questions?
- Open an issue with the `question` label
- Check existing issues and discussions
- Reach out to maintainers for guidance

## Recognition
Contributors will be recognized in:
- Pull request acknowledgments
- Project documentation
- Contributor list (if maintained)

Happy contributing! 🌳
